Audio Transcriber – Bangla Audio to Bangla/English Verbatim Transcription

Problem Statement / Purpose

For qualitative research, accurate transcription and English translation are essential. In collaboration with Development Economist Duncan Webb, funded by Princeton University, we collected over 180 qualitative interviews that required verbatim transcription and translation.

Relying solely on manual transcription would be extremely time- and resource-intensive. To address this challenge, we built a custom solution powered by generative AI models to make transcription more cost-effective and time-efficient.

Technology Used

Frontend: HTML, CSS
Backend: Python (Flask)
API: Gemini 2.5 Pro

Outcome

Typically, one research assistant can complete only one transcription per day, meaning 180 interviews would take approximately 180 days.

With our AI-powered transcriber, a single assistant can complete up to 4 transcriptions per day. While AI does not achieve 100% accuracy, it delivers an estimated 70–80% accuracy, which can be refined by research assistants for higher precision.

This innovation reduces the timeline from 180 days to just 45 days, making the process significantly faster, more cost-effective, and resource-efficient.

Custom STATA Packages – Automating Survey Data Cleaning

Description

As I’m working as a research analyst and my main responsibilities involve to cleaning and analyzing the survey data, I have to go through the same process for every projects, Based on repetitiveness, I have made some custom STATA packages to reduce repetitiveness task while processing the data.

List of Packages

  • exprep
    In survey data collection (e.g., SurveyCTO), repeat groups are often used to capture detailed information across multiple responses. While helpful for data entry, they can be complex to analyze directly in STATA.
    exprep restructures repeat group data into a clean, organized, and analysis-friendly format without altering the original values to improve readability and usability for researchers.

    GitHub Repo
  • exportables
    For reporting, researchers often need large sets of frequency tables. Previously, this required generating tables in STATA, manually copying them into Excel, and formatting them—a repetitive and time-consuming process.
    exportables automates this workflow, allowing users to export thousands of well-formatted tables with a single command in STATA, saving both time and effort.

    GitHub Repo
  • recode_rep, recode_nrep, recode_oth
    During survey data processing, open-ended responses often create challenges:
    • Enumerators may select “Other (specify)” but type an option that already exists in the predefined list.
    • Genuine new responses may appear that need to be integrated into the dataset.
    These packages automate the process of reintegrating misclassified responses back into the main variables and recoding new valid responses into structured categories.
    • recode_oth → Handles open-ended responses in single-select variables.
    • recode_nrep → Handles open-ended responses in multi-select variables (outside repeat groups).
    • recode_rep → Handles open-ended responses in multi-select variables (inside repeat groups).

    recode_oth recode_nrep recode_rep
  • multisplit
    When cleaning survey data, multi-select variables often need corrections such as removing wrongly selected responses or updating answers based on field notes. Traditionally, this required manually editing both the main multi-select variable and all its split dummy variables, which was repetitive and error-prone.
    multisplit automates this process by regenerating the dummy variables directly from the corrected main multi-select field. This ensures accuracy, saves time, and eliminates the need for manual re-splitting after every correction.
